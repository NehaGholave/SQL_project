## 🛍️📊 Retail Sales Data Analysis using SQL
### 📌 Project Title

Retail Sales Data Analysis using SQL

### 🧾 One-Line Summary

📈 An end-to-end SQL data analysis project to clean retail sales data, perform business analysis, and generate meaningful insights using MySQL.

### 📖 Overview

🔍 This project focuses on analyzing retail sales data to understand customer purchasing behavior, product performance, revenue trends, cancellations, returns, and payment preferences. The project includes complete data cleaning and analytical query solving using SQL.

### ❓ Problem Statement

🚨 Retail businesses generate a large amount of transactional data daily. Without proper analysis, it becomes difficult to:

- 📊 Track sales growth and revenue trends
- 👥 Understand customer buying behavior
- 🛍️ Identify top-selling and low-performing products
- 💳 Analyze preferred payment methods
- 🔁 Monitor return and cancellation rates

👉 The goal of this project is to transform raw retail sales data into actionable business insights using SQL.

### 💼 Business Questions Addressed

- 1️⃣ What are the top 5 most selling products by quantity?
- 2️⃣ Which products are most cancelled?
- 3️⃣ What time of day has the highest purchases?
- 4️⃣ Who are the top 5 highest spending customers?
- 5️⃣ Which product categories generate the highest revenue?
- 6️⃣ What is the return/cancellation rate per category?
- 7️⃣ What is the most preferred payment mode?
- 8️⃣ How does age group affect purchasing behavior?
- 9️⃣ What is the monthly and yearly sales trend?
- 🔟 Are certain genders buying more specific product categories?

### 📂 Dataset

- 📁 File Name: sales_store_updated.csv / Excel
- 📝 Description: Retail transactional sales data containing customer, product, pricing, quantity, date, payment, and order status information
- 📄 Type: Structured Dataset

### 🛠️ Tools & Technologies

- 💻 Query Language: SQL
- 🗄️ Database: MySQL
- 🧪 Tool Used: MySQL Workbench
- 🔗 Version Control: Git & GitHub

### 📚 SQL Concepts Used

- 🔹 SELECT, WHERE, ORDER BY
- 🔹 GROUP BY, HAVING
- 🔹 Aggregate Functions – SUM(), COUNT(), AVG(), MAX(), MIN()
- 🔹 CASE Statements
- 🔹 Date Functions – YEAR(), MONTH(), STR_TO_DATE()
- 🔹 String Functions – TRIM(), UPPER()
- 🔹 CTE (Common Table Expression)
- 🔹 Window Functions – ROW_NUMBER()
- 🔹 ALTER TABLE, UPDATE, DELETE

### 🔍 Methodology

#### 🧭 Steps followed in this project:

- 📥 Imported dataset into MySQL
- 🧹 Cleaned data by fixing column names and data types
- ⚠️ Handled duplicates and null values
- 🔄 Standardized inconsistent values (Gender, Payment Mode)
- 📊 Performed SQL-based business analysis
- 💡 Extracted meaningful business insights

### 📊 Key Insights

#### ✨ Key findings from the analysis:

- 📈 Identified monthly and yearly sales growth trends
- 🏆 Found top-selling products by quantity
- 💰 Detected highest revenue generating categories
- 👥 Recognized top spending customers
- 💳 Found most preferred payment methods
- 🔁 Measured cancellation and return percentages
- 🎯 Understood age-wise and gender-wise buying behavior

### 📈 Results

- ✅ Helped understand product demand, customer behavior, and operational performance.
